LIF 神經元 (Digital Implementation)

內部電路架構展開 - 點擊 MUX 查看第三層邏輯

運作邏輯： 輸入電流 (i_syn) 與膜電位 (V_mem) 累加，並扣除漏電 (Leak)。若超過閾值則觸發 Spike 並重置，同時啟動不應期 (Refractory) 保護。

Trace 軌跡邏輯 (Digital Implementation)

STDP 的「短期時間記憶」電路

運作邏輯： 這是 STDP 的「短期記憶」。當 `Pre-Spike` 抵達，對應的 `Trace` 暫存器被設為最大值。在每個時脈週期，`Trace` 的值都會「衰減」（-1 或 >> 1）。這樣 STDP 模組就能透過檢查 Trace 值來判斷 Spike 抵達的「新鮮度」。

STDP 學習單元 (FSM Implementation)

權重更新的「決策大腦」

運作邏輯： 這是一個事件驅動的狀態機。平時處於 `IDLE` 狀態。當 `Post-Spike` 抵達，FSM 進入 `CHECK_LTP` 狀態，檢查所有相關的 Pre-Trace，對「有貢獻」的權重進行增強。當 `Pre-Spike` 抵達，FSM 進入 `CHECK_LTD` 狀態，檢查 Post-Trace，對「無關」的權重進行削弱。

Weight Memory (Digital Implementation)

突觸權重的儲存單元

運作邏輯： 這是儲存 SNN「知識」的地方。在「推論 (Inference)」時，它被當作 **ROM**（唯讀），`Pre-Spike` 作為地址 (Address) 讀取權重 (Data_Out)。在「學習 (Learning)」時，它被當作 **RAM**（可讀寫），`STDP` 模組會提供新的權重 (Data_In) 和寫入訊號 (Write_Enable) 來更新它。

LIF MUX 邏輯 (Next-State Logic)

神經元「重置」與「鎖定」的決策電路

運作邏輯： 這是 MUX (多工器)，一個 2 選 1 的開關。它的決策邏輯是：

  • 當 `SEL` = 0 (非觸發且非不應期)： 選擇 `Input 0`。電路正常運作，將累加後的 `V_int` 送入 `V_mem`。
  • 當 `SEL` = 1 (剛觸發 或 仍在不應期)： 選擇 `Input 1`。電路被「鎖定」，強制將 `V_RESET` (重置電位) 送入 `V_mem`，實現重置與不應期功能。
